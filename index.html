Skit Gubbe

Machine Learning AI

In Skit Gubbe I've experimented with using Machine Learning to develop an AI opponent. I first made a simple opponent that only played the lowest card possible but later I started looking in to Q-Learning AI, which is a type of reinforcement learning where the AI learns by trial and error while playing itself. it plays the game over and over and gradually figures out which actions tend to lead to winning. It does that by saving states to a Q-Table which it then references while playing. If it has not seen that exact position/state before it will save it to the Q-Table and if it's actions was good or bad which it decides through a reward system

Machine Learning Simulation

To teach the Q-Learning AI I used an empty scene in Unity to run millions of simulations where it plays itself and learns the states it then saved to the Q-Table. The training first starts with the AI playing randomly so it learns a lot of states early on. Later as the Epsilon value drops the less and less it playes randomly and instead refers to the states in the Q-Table, making it's winrate go up and up through out the simulation.

JSON Save System (Tiny Train Tales)

Before making this system I relied on Unity's PlayerPrefs function to store data, but PlayerPrefs is designed for simple key-value pairs like settings and not structured game data with nested objects. After adding Cargo and an Inventory System where each cargo object and city had many values to store, I felt it was necessary to add a custom Save System to make it both easier and more reliable. The system works by saving data into SaveData classes that are then stored inside SaveFile classes as lists. It creates a separate JSON file for each category of data like cities, inventory, upgrades and passengers, keeping things organized so only the relevant file needs to be rewritten when something changes. These files are placed locally on the player's computer. Cities are also stored in a dictionary keyed by city name so any system can look up a city instantly rather than scanning through the whole list. SaveToDisk() is called manually at key moments like scene transitions and game exit. When the player starts the game it reads the saved files and loads all the data back in.
